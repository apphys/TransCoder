<h2><a id="user-content-dependencies" class="anchor" aria-hidden="true" href="#dependencies"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Dependencies</h2>
<ul>
<li>Python 3</li>
<li><a href="http://www.numpy.org/" rel="nofollow">NumPy</a></li>
<li><a href="http://pytorch.org/" rel="nofollow">PyTorch</a></li>
<li><a href="https://github.com/fairinternal/TransCoder_release/tree/master/XLM/tools#fastbpe">fastBPE</a> (generate and apply BPE codes)</li>
<li><a href="https://github.com/fairinternal/TransCoder_release/tree/master/XLM/tools#tokenizers">Moses</a> (scripts to clean and tokenize text only - no installation required)</li>

<p>If you run the data preprocessing pipeline, you will have to compile fastBPE. Go in XLM/tools/fastBPE and carry out the steps described in the <a href="https://github.com/facebookresearch/TransCoder/blob/master/XLM/tools/README.md">ReadMe</a>.</p>

<p>The model can be tested on new inputs using the translate.py script.</p>
<p>For instance, <code>python translate.py --src_lang cpp --tgt_lang java --model_path trained_model.pth &lt; input_code.cpp</code> will translate the C++ code contained in input_code.cpp to Java.</p>

<h2><a id="user-content-train-a-new-model" class="anchor" aria-hidden="true" href="#train-a-new-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Train a new model</h2>
<h3><a id="user-content-data-needed" class="anchor" aria-hidden="true" href="#data-needed"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Data needed</h3>
<p>Data you need to pretrain a model with MLM:</p>
<ul>
<li>training data (monolingual): source code in each language  , ex: train.python.pth (actually you have 8 of these <code>train.python.[0..7].pth</code> because data is split on 8 gpu)</li>
<li>test / valid data (monolingual): source code in each language to test perplexity of model , ex: <code>test.python.pth</code> / <code>valid.python.pth</code></li>
</ul>
<p>Data you need to train AE and BT :</p>
<ul>
<li>training data (monolingual): functions standalone in each language, ex: <code>train.python_sa.[0..7].pth</code></li>
<li>test / valid data (monolingual + parallel):
<ul>
<li>monolingual functions to test perplexity of model, ex: <code>test.python_sa.pth</code> / <code>valid.python_sa.pth</code></li>
<li>parallel functions to test the translation model (with BLEU and unit tests), ex: <code>test.python_sa-cpp_sa.pth</code> / <code>valid.python_sa-cpp_sa.pth</code></li>
</ul>
</li>
</ul>
<p>All of these data should be contain in one folder. The path is given in the --data argument.
We provide the parallel test and validation data. See section <a href="https://github.com/fairinternal/TransCoder_release/blob/master/README.md#run-an-evaluation">Run an evaluation</a> and <a href="https://github.com/fairinternal/TransCoder_release/blob/master/README.md#validation-and-test-sets-release">Validation and Test Sets Release</a>. To obtain all the monolingual data (all code/functions // train/test/valid) see the following section.</p>
<p>NB : In our case, the training data is too large to fit on a single machine. Thus, we split it into 8 binarized files and at training, we split the data across 8 GPU. If your training data can fit on a single machine, regroup all your training data into a single file for instance <code>train.python.pth</code></p>
<h3><a id="user-content-downloadpreprocess-data" class="anchor" aria-hidden="true" href="#downloadpreprocess-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Download/preprocess data</h3>
<p>To get the monolingual data, first download cpp / java / python source code from Google BigQuery (<a href="https://cloud.google.com/blog/products/gcp/github-on-bigquery-analyze-all-the-open-source-code" rel="nofollow">https://cloud.google.com/blog/products/gcp/github-on-bigquery-analyze-all-the-open-source-code</a>). To run our preprocessing pipeline, you need to donwlaod the raw source code on your machine in json format, and put each programming language in a dedicated folder. A sample of it is given in data/test_dataset. The pipeline extracts source code from json, tokenizes it, extracts functions, applies bpe, binarizes the data and creates symlink with appropriate name to be used directly in XLM. The folder that ends with .XLM-syml is the data path you give for XLM traning. You will have to add the test and valid parallel we provide in "Run an evaluation" data to that folder.</p>
<p>To test the pipeline run <code>pytest preprocessing/test_preprocess.py</code>, you will see the pipeline output in data/test_dataset folder.</p>
<p>To run the pipeline (either locally or on remote machine ), command example:</p>
<pre><code>python -m preprocessing.preprocess 
absolute_path_to_TranscCoder/data/test_dataset # path to the root folder where you have the json
--lang1 java # languages to prepocess
--lang2 python #
--lang3 cpp # can be None if you want to preprocess only 2 languages
--keep_comments True # True if you want to keep code comments in you code, False to remove them
--bpe_train_size 0 # Set the size of the training data subset on which the bpe codes are trained. 0 -&gt; parameter disabled and all training data are used
--test_size 10 # size of test/validation sets , usually 1000, here 10 to test the command on the json samples
--local True # True if you want to launch the pipeline locally , False to launch on remote machine. In that case it uses submitit
</code></pre>
<p>If you want to preprocess another programming language, you have to implement the functions tokenize_newlang, detokenize_newlang, extract_function_newlang, get_function_name_newlang in preprocessing/src/code_tokenizer.py and run the pipeline with newlang.</p>
<p>NB: If you run the pipeline for cpp/java/python with --keep-comments True, you dont need to train the bpe codes and vocab, they are provided in data/bpe.cpp-java-python.with_comments. In the folder where have your json folders, you simply have to add a folder cpp-java-python.with_comments and copy the codes and vocab files. The pipeline will see it and pass the BPE traing step.</p>
<h3><a id="user-content-pretrain-a-model-with-mlm" class="anchor" aria-hidden="true" href="#pretrain-a-model-with-mlm"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Pretrain a model with MLM</h3>
<p>Example:</p>
<pre><code>python XLM/train.py 

--n_heads 8 
--bt_steps '' 
--max_vocab 64000 
--word_mask_keep_rand '0.8,0.1,0.1' 
--word_blank 0

<h2><a id="user-content-validation-and-test-sets-release" class="anchor" aria-hidden="true" href="#validation-and-test-sets-release"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Validation and Test Sets Release</h2>
<p>We release our validation and test dataset. You can download the raw data <a href="https://dl.fbaipublicfiles.com/transcoder/TransCoder_tokenized_test_set_functions.zip" rel="nofollow">here</a>.</p>
<p>The format of each line in each file is <code>&lt;FUNCTION_ID&gt; | &lt;function&gt;</code>. The function are tokenized. You can detokenize them with the script preprocessing/detokenize.py. You can extract the function id and use it to find the corresponding test script in <code>data/evaluation/geeks_for_geeks_successful_test_scripts/&lt;language&gt;</code> if it exists.</p>
<p>For instance, for the line <code>COUNT_SET_BITS_IN_AN_INTEGER_3 | &lt;function&gt;</code> in the file test.cpp.shuf.valid.tok, the corresponding test script can be found in <code>data/evaluation/geeks_for_geeks_successful_test_scripts/cpp/COUNT_SET_BITS_IN_AN_INTEGER_3.cpp</code>.
If the script is missing, it means there was an issue with our automatically created tests for the corresponding function.</p>
<p>The code generated by your model can be tested by injecting it where the <code>TO_FILL</code> comment is in the test script.</p>
<h2><a id="user-content-little-guide-to-download-github-from-google-big-query" class="anchor" aria-hidden="true" href="#little-guide-to-download-github-from-google-big-query"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Little guide to download Github from Google Big Query</h2>
<ul>
<li>Create a Google platform account ( you will have around 300 $ given for free , that is sufficient for Github)</li>
<li>Create a Google Big Query project here</li>
<li>In this project, create a dataset</li>
<li>In this dataset, create one table per programming language. The results of each SQL request (one per language) will be stored in these tables.</li>
<li>Before running your SQL request, make sure you change the query settings to save the query results in the dedicated table (more -&gt; Query Settings -&gt; Destination -&gt; table for query results -&gt; put table name)</li>
<li>Run your SQL request (one per language and dont forget to change the table for each request)</li>
<li>Export your results to google Cloud :
<ul>
<li>In google cloud storage, create a bucket and a folder per language into it</li>
<li>Export your table to this bucket ( EXPORT -&gt; Export to GCS -&gt; export format JSON , compression GZIP)</li>
</ul>
</li>
<li>To download the bucket on your machine, use the API gsutil:
<ul>
<li>pip install gsutil</li>
<li>gsutil config -&gt; to config gsutil with your google account</li>
<li>gsutil -m cp -r gs://name_of_bucket/name_of_folder . -&gt; copy your bucket on your machine</li>
</ul>
</li>
</ul>
<p>Example of query for python :</p>
<pre><code>SELECT 
 f.repo_name,
 f.ref,
 f.path,
 c.copies,
 c.content
FROM `bigquery-public-data.github_repos.files` as f
  JOIN `bigquery-public-data.github_repos.contents` as c on f.id = c.id
WHERE 
  NOT c.binary
  AND f.path like '%.py'
</code></pre>
